휴고 블로그 만들기

이 글은 Hugo라는 정적 사이트 생성기(Static site generator)를 이용해 깃헙 페이지로 블로그를 개설하는 방법을 소개한다.

Hugo 설치

휴고 공식 사이트에서 설치 방법을 참고하여 사용 중인 OS에 맞게 설치한다.

바이너리 파일을 받거나, 패키지 매니저로 설치할 수 있다.

여기서는 Windows에서 Chocolatey 패키지 매니저를 통해 설치한다.

choco install hugo -confirm

설치가 완료되었다면 hugo 커맨드를 사용할 수 있다.

hugo version 커맨드를 입력하여 버전을 확인해볼 수 있다.

hugo version
hugo v0.102.1-e9c3ce16abc391ee5be38f88b0499d0e8e19104c windows/amd64 BuildDate=2022-08-29T12:58:13Z VendorInfo=gohugoio

블로그 만들기

초기화

적당한 디렉토리에 hugo new site blog 커맨드를 입력한다.

그러면 blog 디렉토리에 블로그에 필요한 파일들이 초기화된다.

테마 선택

글을 작성하기 전에, 테마를 먼저 골라야한다.

휴고 사이트에서 제공하는 테마 목록에서 마음에 드는 것을 골라 Git 저장소 주소를 복사해둔다.

여기서는 tale 테마를 선택하였다.

테마 설정

다음과 같이 깃에 서브모듈로서 가져온다.

git submodule add https://github.com/EmielH/tale-hugo themes/tale

휴고는 themes 디렉토리 안에 테마를 관리하므로, themes/tale 디렉토리로 가져왔다.

블로그 디렉토리의 config.toml 파일을 열어, 테마를 설정한다.

theme = 'tale'

글 작성하기

파일 만들기

블로그에 올릴 글은 마크다운으로 작성할 수 있다. (마크다운 문법은 이 페이지를 참고하자.)

다음 커맨드로 마크다운 파일을 초기화한다.

hugo new posts/hello.md

그러면 제목과 날짜, 초안 여부가 다음과 같이 생성된다.

---
title: "Hello"
date: 2022-08-30T22:30:00+09:00
draft: true
---

---으로 감싼 이 부분은 휴고에서 Front matter라고 부르는 부분으로, 각종 메타데이터를 기록할 수 있다.

이 부분은 마크다운 문법이 아닌 yaml이나 toml, json으로 작성한다는 점을 참고하자.

이 아래에 원하는 글을 마크다운으로 작성한다.

로컬에서 렌더링 결과 보기

hugo server -D 커맨드를 입력하자.

그러면 다음과 같은 출력을 볼 수 있다.

Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)

웹 브라우저에 localhost:1313 주소로 들어가면 렌더링된 블로그를 볼 수 있다.

이 상태에서 글을 수정하면 웹 페이지가 바로 업데이트 된다. 이 커맨드를 실행해두고 글을 작성해보자.

글 완성하기

방금 커맨드에서 -D는 초안인 글까지 렌더링한다. (즉 Front matter에 draft 항목이 true인 글)

글을 전부 작성했으면 draft 항목을 없애거나 false로 하자.

깃헙 페이지로 배포하기

필요한 저장소

다음과 같은 두 개의 저장소가 필요하다.

  1. 방금 만든 블로그 파일들을 위한 저장소

    이 저장소 이름은 아무것이나 상관 없지만, 여기서는 wcho21.github.io-hugo라고 지었다.

  2. 렌더링한 파일을 위한 저장소

    깃헙 페이지로 배포하려면, 깃헙 아이디를 이용해 <id>.github.io와 같은 이름으로 만들어야 한다.

    그러면 실제로 <id>.github.io 주소로 접속하여 렌더링된 블로그를 볼 수 있게 된다.

    여기서는 wcho21.github.io라는 이름으로 만들었다.

서브모듈 만들기

렌더링된 블로그는 public 디렉토리에 만들어진다.

그러므로 <id>.github.io 저장소를 다음과 같이 서브모듈로 추가한다.

git submodule add <id>.github.io public

렌더링

hugo 커맨드를 실행하면 public 디렉토리에 결과가 생성된다.

배포

public 디렉토리로 이동해서 커밋하고 푸시한다.

다음은 예시이므로 커밋 메시지는 다르게 해도 상관없다.

cd public
git commit -m "create a hugo blog"
git push

이제 <id>.github.io 주소로 접속하면 블로그를 볼 수 있다.

블로그 디렉토리도 커밋하고 푸시한다.

cd ..
git commit -m "create a hugo blog"
git push
