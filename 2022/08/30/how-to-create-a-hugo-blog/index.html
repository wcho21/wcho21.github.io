DEVLOG W

    • 휴고 블로그 만들기

    2022-08-30

    Hugo라는 정적 사이트 생성기를 이용해 블로그를 개설하는 방법을 소개한다.

    TL;DR

    1. Hugo를 로컬에 설치한다.

    2. 블로그 테마를 고르고 글을 작성한다.

    3. GitHub Pages 서비스를 이용해 블로그를 개설한다.

    Hugo 설치

    공식 문서의 설치 방법을 참고하여 사용 중인 OS에 맞게 설치하자.

    여기서는 Windows에서 Chocolatey 패키지 매니저를 통해 설치한다.

    choco install hugo -confirm

    두 개의 버전

    hugo 대신 hugo-extended 버전을 설치하면 휴고가 자체적으로 Sass를 지원한다.

    블로그 스타일 작업시 Sass를 사용하고 싶다면 이 버전을 고려해보자.

    버전 확인

    이제 hugo 커맨드를 사용할 수 있다. 다음 커맨드로 버전을 확인해보자.

    $ hugo version

    hugo v0.102.1-e9c3ce16abc391ee5be38f88b0499d0e8e19104c windows/amd64 BuildDate=2022-08-29T12:58:13Z VendorInfo=gohugoio

    블로그 만들기

    적당한 위치에서 hugo new site <directory> 커맨드를 입력하자.

    해당 <directory>에 파일이 초기화된다.

    테마 선택

    먼저 테마를 먼저 골라야한다.

    휴고에서 제공하는 테마 목록에서 마음에 드는 것을 골라 Git 저장소 주소를 복사해두자.

    여기서는 tale 테마를 예로 든다.

    테마 설정

    다음과 같이 깃에 서브모듈로서 가져온다.

    $ git submodule add https://github.com/EmielH/tale-hugo themes/tale

    휴고는 themes 디렉토리 안에 테마를 관리하므로, themes/tale 디렉토리로 가져온다.

    이제 블로그 디렉토리의 config.toml 파일을 열어, 테마를 설정한다.

    theme = 'tale'

    포스트 작성하기

    블로그에 올릴 글은 마크다운으로 작성한다.

    다음 커맨드로 마크다운 파일을 하나 새로 만들자.

    $ hugo new posts/hello.md

    그러면 제목과 날짜, 초안 여부가 다음과 같이 생성된다.

    ---
title: "Hello"
date: 2022-08-30T22:30:00+09:00
draft: true
---

    ---으로 감싼 이 부분은 휴고에서 Front matter라고 부르는 부분으로, 제목을 비롯한 메타 데이터가 YAML 문법으로 작성된다.

    이 아래 부분에 마크다운으로 원하는 포스트를 작성하자.

    로컬에서 결과 보기

    hugo server -D 커맨드를 입력하면 다음과 같은 주소가 출력된다.

    Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)

    이제 웹 브라우저에 localhost:1313 주소로 들어가면 작성한 글을 확인할 수 있다.

    Hugo is watching you

    이 커맨드를 실행한 상태에서 글을 수정하면 브라우저에서 실시간으로 업데이트 된다.

    -D 옵션

    hugo server -D 커맨드는 초안인 글까지 빌드한다.

    글을 전부 작성했으면 Front matter에서 draft 항목을 없애거나 false로 하자.

    그러면 -D 옵션 없이 빌드할 수 있다.

    GitHub Pages로 배포하기

    이제 로컬이 아니라 모든 사람이 볼 수 있게 빌드하고 배포할 것이다.

    hugo 커맨드를 실행해보자. 그러면 public 디렉토리에 정적 페이지가 빌드된다.

    이 결과를 GitHub Pages로 배포할 것이다.

    필요한 저장소

    GitHub 다음과 같은 두 개의 저장소를 만들자.

    1. public 디렉토리를 위한 저장소

      빌드 결과를 위한 저장소이다.

      GitHub 유저네임을 이용해 <username>.github.io와 같은 이름으로 저장소를 만들자.

      이런 이름으로 만들면, 저장소에 푸시된 것을 GitHub Pages 서비스가 <username>.github.io 주소로 자동 배포해준다.

      여기서는 wcho21.github.io라는 이름으로 만들었다.

    2. 블로그 디렉토리를 위한 저장소

      아무것이나 상관 없지만, 여기서는 wcho21.github.io-hugo라고 만들었다.

    서브모듈 만들기

    블로그 저장소와 빌드 결과 저장소를 따로 관리하기 위함이다.

    블로그 디렉토리에서 <username>.github.io 저장소를 다음과 같이 서브모듈로 추가한다.

    $ git submodule add <username>.github.io public

    배포

    public 디렉토리로 이동해서 커밋하고 푸시한다.

    $ cd public
$ git commit -m "create a hugo blog"
$ git push

    이제 <username>.github.io 주소로 접속하면 블로그를 볼 수 있다.

    블로그 디렉토리도 커밋하고 푸시한다.

    $ cd ..
$ git commit -m "create a hugo blog"
$ git push
